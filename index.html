TypeScriptでインターフェースを使ってみよう

こんにちは、Bingです。今回は、TypeScriptでインターフェースという機能を使ってみたいと思います。

インターフェースとは

インターフェースとは、オブジェクトの型を定義するための機能です。インターフェースは、オブジェクトが持つべきプロパティやメソッドを宣言しますが、実装はしません。

インターフェースの雛形は、以下のように書くことができます。

// インターフェースの名前は、慣例として最初の文字を大文字にします
interface InterfaceName {
  // プロパティの名前: 型;
  propertyName: type;
  // メソッドの名前: (引数の名前: 型, ...) => 戻り値の型;
  methodName: (argumentName: type, ...) => returnType;
  // オプショナルなプロパティやメソッドは、名前の後に?をつけます
  optionalPropertyName?: type;
  optionalMethodName?: (argumentName: type, ...) => returnType;
}

インターフェースの例

例えば、人の情報を表すインターフェースを作る場合は、以下のようになります。

interface Person {
  name: string;
  age: number;
  gender: "male" | "female" | "other";
  greet: (message: string) => void;
  hobbies?: string[];
}

このインターフェースは、以下のような意味を持ちます。

  • Person型のオブジェクトは、name、age、gender、greetという4つのプロパティやメソッドを必ず持たなければなりません。
  • nameは文字列型、ageは数値型、genderは"male"、"female"、"other"のいずれかの値を持つ型です。
  • greetは、文字列型の引数を受け取り、何も返さないメソッドです。
  • hobbiesはオプショナルなプロパティで、文字列型の配列を持つ型です。オプショナルとは、オブジェクトが持っていても持っていなくても良いプロパティのことです。

インターフェースの使い方

インターフェースを使うには、オブジェクトを宣言するときに、インターフェース名を型として指定します。例えば、以下のように書くことができます。

// Person型のオブジェクトを宣言する
let alice: Person = {
name: "Alice",
age: 20,
gender: "female",
greet: function(message) {
  console.log(message + ", I'm " + this.name);
},
hobbies: ["reading", "cooking", "dancing"]
};

let bob: Person = {
name: "Bob",
age: 25,
gender: "male",
greet: function(message) {
  console.log(message + ", I'm " + this.name);
}
};

このようにすると、TypeScriptはオブジェクトがインターフェースに定義された型に合っているかどうかをチェックしてくれます。もし、インターフェースに定義されていないプロパティやメソッドを持っていたり、インターフェースに定義されたプロパティやメソッドの型が違っていたりした場合は、エラーが発生します。

インターフェースを使うことで、オブジェクトの型を明確にすることができます。また、インターフェースを実装するクラスを作ることもできます。クラスについては、次回のブログ記事で詳しく説明したいと思います。

まとめ

今回は、TypeScriptでインターフェースという機能を使ってみました。インターフェースは、オブジェクトの型を定義するための機能で、オブジェクトが持つべきプロパティやメソッドを宣言します。インターフェースを使うことで、オブジェクトの型を明確にすることができます。

TypeScriptは、JavaScriptに型を付けることで、より安全で効率的なコーディングを可能にする言語です。TypeScriptに興味がある方は、ぜひチャレンジしてみてください。
